\chapter{Deviations And Conclusion}

\section{Main Deviations From Assignment 1}

The final implementation evolved from the Assignment 1 design in several practical ways:

\begin{itemize}
  \item FastAPI became the authoritative backend runtime.
  \item The realtime relay remained a separate Node/Yjs service rather than being absorbed into FastAPI.
  \item SQLite was retained for portability and reproducibility.
  \item Local password reset replaced external email delivery in the local runtime.
  \item AI provider integration supports both deterministic stub mode and LM Studio-backed mode.
\end{itemize}

These deviations are documented in the repository so that the final implementation and the earlier design
can be compared directly and transparently.

\section{Conclusion}

The final Assignment 2 implementation covers the required baseline and multiple bonus-tier elements while
keeping security, collaboration, and AI behaviour coherent across the repository. The system supports a
local run path, automated tests, a report package, and a clear mapping between architecture, code, and
runtime behaviour.

The final repository, docs, tests, and runtime now describe the same delivered system. That consistency is
the main reason the implementation is suitable for submission.
